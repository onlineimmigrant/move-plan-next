## Choosing Your AI Models

One of Coded Harmony's most powerful features is its flexible AI system. Let's set up the AI models that will power your business.

### Understanding AI Models

**What are AI Models?**
AI models are the "brains" behind various intelligent features in Coded Harmony. Each model is trained for specific tasks like writing content, answering questions, or analyzing data.

**Why Multiple Models?**
Different AI models excel at different tasks:
- **GPT-4**: Best for creative writing, complex reasoning
- **Claude 3**: Excellent for long-form content, analysis
- **Grok**: Great for real-time information, conversational tone
- **Gemini**: Strong at multimodal tasks, data analysis
- **DeepSeek**: Cost-effective, good for routine tasks

### Accessing AI Model Management

1. **Navigate to AI Models**
   - From dashboard, click **"AI Models"** in sidebar
   - Or go to: **Admin â†’ AI â†’ Management**

2. **You'll See Three Tabs:**
   - **Models**: Your organization's AI models
   - **Add Model**: Create custom AI agents
   - **System**: Browse and enable global models

### Step 1: Enable System Models (Recommended First Step)

System models are pre-configured and ready to use immediately.

**To Enable a System Model:**

1. **Go to "System" Tab**
   - See all available system models filtered by your organization type
   - Models show: name, description, pricing tier requirement, token limits

2. **Browse Available Models**
   
   **Content Creation Models:**
   - **Blog Writer AI**: Generates blog posts and articles
   - **Product Description AI**: Creates compelling product copy
   - **Email Composer AI**: Drafts professional emails
   - **Social Media AI**: Creates posts for social platforms
   
   **Customer Support Models:**
   - **Support Agent AI**: Answers customer questions
   - **Ticket Responder AI**: Suggests responses to support tickets
   - **FAQ Generator AI**: Creates FAQ content from documentation
   
   **Specialized Models (by industry):**
   - **Legal Document AI**: Contract review, legal drafting (for law firms)
   - **Course Creator AI**: Lesson plans, quiz generation (for educators)
   - **Patient Communication AI**: Healthcare communications (for medical)
   - **Immigration Document AI**: Visa application assistance (for immigration services)

3. **Enable Models for Your Team**
   - Click **"Enable"** button on any model
   - Model is instantly available to all users in your organization
   - Set user quotas (optional): limit how many tokens each user can consume

4. **Try a Model**
   - Enabled models appear in your "Models" tab
   - Click on any model to test it
   - Enter a sample prompt and see results

**ğŸ’¡ Recommended Starting Models:**
- **Content Writer AI**: Essential for blog posts and web pages
- **Support Agent AI**: Powers your help center chatbot
- **Email Composer AI**: Speeds up customer communications

### Step 2: Configure Organization Models (Optional)

Create custom AI models tailored to your business needs.

**When to Create Custom Models:**
- You need AI trained on your specific brand voice
- You have unique workflows or processes
- You want to combine multiple capabilities
- You need industry-specific compliance rules

**To Create a Custom Model:**

1. **Go to "Add Model" Tab**

2. **Fill in Model Details:**
   
   **Basic Information:**
   - **Name**: Descriptive name (e.g., "Acme Legal Client Intake Assistant")
   - **Role**: What this AI does (e.g., "Client intake specialist")
   - **Icon**: Choose an icon to represent this model
   
   **AI Configuration:**
   - **Base Model**: Choose foundation (GPT-4, Claude, etc.)
   - **System Message**: Instructions that guide the AI's behavior
     ```
     Example for law firm:
     "You are a client intake specialist for Acme Legal Services,
     a firm specializing in family law and estate planning.
     Your tone is professional yet warm. You help potential clients
     understand our services and gather initial information.
     Always mention our free 30-minute consultation."
     ```
   - **Max Tokens**: Response length limit (100-2000)
   - **Temperature**: Creativity level (0=focused, 1=creative)
   
   **API Configuration (Advanced):**
   - **API Key**: Your API key for the AI provider
   - **Endpoint**: API URL (usually pre-filled)
   - **Custom Parameters**: JSON configuration for advanced users
   
   **Access Control:**
   - **Who can use this?** All users, Admins only, Specific roles
   - **Token Limit per User**: Daily/weekly/monthly quota
   - **Active Status**: Enable/disable the model

3. **Define Tasks (Optional but Powerful)**
   
   Tasks are predefined workflows that make AI even more useful:
   
   **Example Task: "Generate Client Welcome Email"**
   ```json
   {
     "name": "Client Welcome Email",
     "description": "Create personalized welcome email for new clients",
     "prompt_template": "Write a welcome email for {client_name} who hired us for {case_type}. Include information about {attorney_name} and next steps.",
     "variables": ["client_name", "case_type", "attorney_name"],
     "output_format": "email"
   }
   ```
   
   Users can then fill in the variables and get consistent, branded output.

4. **Test Your Model**
   - Use the test interface to validate behavior
   - Try different prompts and scenarios
   - Adjust system message if results aren't ideal

5. **Save and Enable**
   - Click "Save Model"
   - Toggle "Active" to make it available
   - Model appears in "Models" tab for all users

**ğŸ’¡ Pro Tip**: Start with a broad model and create specialized versions as needs emerge.

### Step 3: Set User Quotas and Policies

Control AI usage to manage costs and ensure fair access.

**Organization-Wide Settings:**

1. **Navigate to**: Admin â†’ AI â†’ Settings

2. **Configure Token Budgets:**
   - **Monthly Organization Budget**: Total tokens for all users
   - **Default User Quota**: Tokens per user per month
   - **Rollover Policy**: Unused tokens carry over or reset?
   - **Overage Handling**: Block usage or allow with notification

3. **Usage Alerts:**
   - Email admin at 80% of quota consumed
   - Notify users approaching their limit
   - Weekly usage reports

4. **Model Access by Role:**
   - Admins: All models, unlimited
   - Content Managers: Writing models, higher quota
   - Support Agents: Support models, medium quota
   - Standard Users: Basic models, lower quota

**Per-User Overrides:**

For specific users who need more/less access:
- Go to: Admin â†’ Users â†’ [Select User]

- Enable/disable specific models

### Step 4: Monitor AI Usage

Track how AI is being used across your organization.

**AI Analytics Dashboard:**
- Total tokens consumed (daily, weekly, monthly)
- Usage by user, by model, by department
- Most popular models
- Cost tracking (if applicable)
- Performance metrics (response time, success rate)

**Optimization Tips:**
- Models with low usage can be disabled
- High usage indicates valuable modelsâ€”increase quotas
- Monitor for unusual patterns (potential misuse)
- Review and adjust based on business priorities

### AI Model Best Practices

**DO:**
- âœ… Start with 2-3 system models, expand gradually
- âœ… Test models thoroughly before rolling out to team
- âœ… Create task-based workflows for consistency
- âœ… Monitor usage and adjust quotas based on need
- âœ… Train your team on effective AI prompting
- âœ… Use specialized models for specialized tasks

**DON'T:**
- âŒ Enable all models at once (overwhelming for users)
- âŒ Set unlimited quotas without monitoring
- âŒ Use creative models for factual content
- âŒ Ignore user feedback on model performance
- âŒ Skip testing custom models before deployment
- âŒ Use AI for sensitive data without proper safeguards
